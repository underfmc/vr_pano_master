# Следующие шаги после MVP

## 1. Карта выбора дома

Frontend: MapLibre/Leaflet/Yandex Maps JS API.

Пользователь выбирает точку → backend вызывает:

```bash
python master.py run-auto --project <id> --lat <lat> --lon <lon> --levels <n>
```

Дальше UI показывает контур ближайшего OSM-здания и просит подтвердить.

## 2. Улучшение 3D-сцены

Текущая PBR-сцена работает, но можно улучшить:

- **Более детальные здания**: LOD-система (ближние здания детальнее)
- **Реальные текстуры**: Photo-scanned PBR materials вместо procedural
- **Транспорт**: процедурные машины на дорогах и парковках
- **Люди**: instanced silhouettes на тротуарах
- **Сезонность**: зимний/летний вариант (снег, листва)
- **Время суток**: golden hour, вечерние огни в окнах

## 3. WebVR слой

После получения `output/aerial_panorama_360.jpg` нужно собрать Three.js viewer:

- Сферический skybox из equirectangular
- Плоскость SVG/PNG-планировки над крышей главного дома
- Переключение этажей/секций
- POI-маркеры по координатам из OSM
- Карточки квартир и форма заявки
- Адаптивный дизайн (desktop + mobile VR)

## 4. Yandex Панорамы

Сейчас `fetch-yandex-pano` скачивает панорамы с нескольких точек вокруг здания. Для коммерческого продукта:

- Проверить лицензию/разрешение Яндекса
- Альтернативы: собственная съёмка, Mapillary, лицензированные источники

## 5. AI-улучшения

- **Flux.1 / SDXL** вместо SD1.5 для AI-polish
- **ControlNet Depth**: использовать depth pass из Blender для точной геометрии
- **Inpainting**: автоматическое исправление артефактов
- **Upscaling**: Real-ESRGAN для 8K+ панорам

## 6. Автоматизация

- **CI/CD**: автоматический рендер при обновлении OSM-данных
- **Batch processing**: рендер нескольких проектов параллельно
- **API**: REST API для интеграции с CRM/сайтами застройщиков
- **Очередь задач**: Celery/RQ для обработки batch-запросов

## 7. Монетизация

- SaaS для застройщиков и агентств недвижимости
- API для интеграции с существующими платформами
- White-label решения
- Подписка на обновление панорам (при изменении района)
